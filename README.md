## Sarrera
Gu 7POINTS.net gara, Migel altuna LHII ikastetxean sorturiko taldea. Proiektu hau, gure enpresak unibertsitateko ikasleen artean antzeman duen beharrizan zehatz bati erantzuteko jaio da.

Zehazki, pisu partekatuetan bizi diren ikasleek eguneroko kudeaketan izaten dituzten zailtasunak identifikatu ditugu: etxeko zereginak antolatzea, gastuak modu garden batean banatzea eta 
elkarbizitzaren koordinazio orokorra. Gaur egungo tresnek ez dituzte behar hauek modu bateratuan asetzen. Hori dela eta, guk, talde bezala, hutsune hori betetzeko web aplikazio bat 
garatzea erabaki dugu, ikasleen bizi-kalitatea eta antolakuntza hobetzeko asmoz.

## Helburuak
<p align="justify">
 Erronka honen helburu nagusia, pisu partekatuen kudeaketarako plataforma bakar eta bateratu bat sortzea izan da. Aplikazio honek ikasleen elkarbizitza erranzten du, gatazkak murriztuz eta ordena eskainiz.
</p>

<p align="justify">
 Bertan, erabiltzaileek etxebizitzaren funtzionamendua kontrolatzeko aukera izango dute. Funtzio nagusien artean, etxeko zereginen banaketa eta gastuen edo ordainketen kontrola nabarmentzen dira. 
  Horretarako, sistema bat garatuko da non informazio guztia argi eta eskuragarri egongo den pisu kide guztientzat.
</p>

<p align="justify">
  Alderdi teknikoari dagokionez, soluzio sendo eta eskalagarri bat eraikitzeko teknologia aurreratuak erabili ditugu.
  Frontend-a garatzeko <strong>React</strong> framework-a aukeratu da, interfaze arin eta dinamiko bat lortzeko. 
  Backend-aren logika eta API zerbitzuak kudeatzeko, berriz, <strong>Laravel</strong> erabili dugu. 
  Eta bukatzeko, sistemaren bihotza, datuen bilketarako eta finantza-kudeaketarako, <strong>Odoo</strong> izan da, honek ematen duen potentziala aprobetxatuz.
</p>

<p align="justify"> 
  Azpiegitura eta garapen ingurunea ere ez dira atzean geratu. 
  Proiektua <strong>AWS</strong> ingurunean hedatzea aurreikusi da segurtasuna eta eskalagarritasuna bermatzeko, eta garapen taldearen lana bateratzeko <strong>Docker</strong> erabili dugu. 
</p>

<p align="justify">
  Azkenik, diseinuaren kalitatea eta irisgarritasuna lehenetsi dira. Webguneak itxura garbia eta erakargarria izateaz gain, erabiltzeko erraza izan behar du edozein profilentzat. 
  Horregatik, <strong>WCAG</strong> (Web Content Accessibility Guidelines) gidalerroak jarraitu dira. Helburu hori bermatzeko, <strong>EqualWeb</strong> tresna erabili dugu, aplikazioa pertsona guztientzat irisgarria eta eskuragarria dela ziurtatuz.
</p>

<p align="justify">
  Beraz, helburua pisu partekatuen kudeaketarako webgune bat sortzea izan da, aipaturiko atal guztiak errespetatuz.
</p>

## Instalazio prozezua

Kasu honetan hurrengoak izango dira instalatzerako orduan **kontuan izan** behar ditugun gauzak:

1. **PHP:**
   - Gure ordenagailuaren PHP bertsioa 8.2 edo altuagoa izan behar da.
2. **Composer:**
   - Gure composer bertxioa 2.0 izan behar da.
3. **Node.js:**
   - Gure Node.js-en bertsioa 18 izan behar da.
4. **NPM edo Yarn:**
   - Gure kasuan NPM instalatzea gomendatzen dugu. Halaber, Yarn ere erabili daiteke.
5. **SQL Lite:**
   - Datu base bezala, SQL Lite instalatzea gomendatzen dugu. Halaber, Laravelekin bateragarria den edozein datu base erabili daiteke.
6. **Git:**
   - Git-eko biltegiarekin lan egiteko, Git instalatu behar da.
7. **VSCode:**
   - Gure kasuan VSCode instalatzeko gomendatzen dugu IDE bezala.

### Instalazioa
Gauza guzti hauek kontuan izanik, orain nola instalatu azalduko dugu.
#### 1. Pausua: PHP

<p align="justify">
Lehenik PHP instalatuta dugun begiratu behar dugu. Horretarako, gure <i>C:</i> Karpetara joan behar gara. Bertan, <i>php8.2</i> karpeta aurkitu behar dugu. Hori edo altuago bat agertzen bazaizu, jada PHP egokia duzu. Bestalde, <i>php4.3</i> edo <i>php5.4</i> bezalakoak agertze bazaizkizu, PHPren bertsio zaharkitua duzu, eta ezingo duzu Laravelekin erabili. Horretarako, desinstalatu egin behar duzu. Karpeta ezabatu ezkero nahikoa da.
</p>


Jarraian, instalatu gabea baduzu, Powershell zabaldu eta hurrengoak idatzi:
Lehenik PHP 8.2 instalatuko dugu:
```
Invoke-WebRequest -Uri "https://windows.php.net/downloads/releases/archives/php-8.2-Win32-vs16-x64.zip" -OutFile "$env:USERPROFILE\Downloads\php82.zip"
```
Ondoren, PHP-rentzako karpeta sortu eta bertan zabalduko dugu:
```
New-Item -ItemType Directory -Path "C:\php82" -Force
Expand-Archive -Path "$env:USERPROFILE\Downloads\php82.zip" -DestinationPath "C:\php82"
```
Jarraian, PHP-a konfiguratuko dugu sistemaren PATH atalean agertzeko:
```
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\php82", [EnvironmentVariableTarget]::Machine)
```
PHP-a egoki instalatu den jakiteko:
```
php -v
```
Hurrengoa edo antzekoa jartzen badizu, egoki instalatu da:
```
PHP 8.2.x (cli) (built: ...)
```
Azkenik, Laravelekin erabiltzeko konfiguratuko dugu. Honetarako:
```
Copy-Item "C:\php82\php.ini-development" "C:\php82\php.ini"
notepad C:\php82\php.ini
```
Honek php.ini zabalduko dizu. Bertan hurrengoa bilatu eta hasieran duten ```;``` guztiak ezabatu
```
;extension=curl
;extension=mbstring
;extension=bcmath
;extension=xml
;extension=mysqli
;extension=openssl
;extension=pdo_mysql
;extension=fileinfo
```
Azkenik gorde eta jada PHP egoki instalatuta eta konfiguratuta izango duzu.
 
#### 2. Composer

<p align="justify">
Composerra PHP-rekin lan egiteko beharrezkoa den dependentzien kudeaketarako balio duen kudeatzaile bat da. Hau gabe, ezin da lan egin Laravelen. Composerra instalatuta baduzu, hurrengo komandoa jarri powershellen bertsioa jakiteko:
</p>

```
composer -V
```
Hurrengoa jartzen badizu, egoki instalatuta duzu.
```
Composer version 2.x.x ...
```
Ez bazaizu agertzen edo bertsio desegokia baduzu, hurrengoa jarri daukazun composerra ezabatzeko:
```
Remove-Item -Recurse -Force "C:\ProgramData\ComposerSetup" -ErrorAction SilentlyContinue
```
Azkenik, instalatu composer bertsio egokian:
```
Invoke-WebRequest -Uri "https://getcomposer.org/installer" -OutFile "$env:USERPROFILE\Downloads\composer-setup.php"
php "$env:USERPROFILE\Downloads\composer-setup.php" --install-dir=C:\php82 --filename=composer
```
Berriz konprobatu composerra, bertsio egokian instalatu den jakiteko.
 
#### 3. Node.js eta NPM
<p align="justify">
Node.js gure JS kodeak nabigatzailetik kanpo exekutatzeko balio du. Hau da, zerbitzari aldeak JS-eko kode atalekin egingo lirateke. Bestalde, NPM-ek <i>Node Package Manager</i> esan nahi du, eta Node.js-en paketean kudeatzeko balio du.
</p>

Bertsioak jakiteko:
```
node -v
npm -v
```
Hurrengoa bezalakoa ematen badizu, egoki instalatuta daude.
```
v18.20.2
10.8.2
```
Egoki instalatu ez badira edo ez badituzu, ezabatu egin behar dituzu. Honetarako:
```
Remove-Item -Recurse -Force "C:\nodejs"
```


Ondoren, instalatu egin behar dituzu:
Node.js-entzako:
```
Invoke-WebRequest -Uri "https://nodejs.org/dist/latest-lts/win-x64/node.exe" -OutFile "C:\nodejs\node.exe"
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\nodejs", [EnvironmentVariableTarget]::Machine)
```
NPM-rentzako:
```
Invoke-WebRequest -Uri "https://registry.npmjs.org/npm/-/npm-10.8.2.tgz" -OutFile "$env:USERPROFILE\Downloads\npm.tgz"
tar -xzf "$env:USERPROFILE\Downloads\npm.tgz" -C "C:\nodejs"
```
Berriz begiratu egoki instalatu dituzun.
 
#### 4. VSCode
VSCode guk erabili dugun IDE-a da. Nabigatzailetik bertatik instalatu dezakezu [.exe](https://code.visualstudio.com/Download) bezala, ondoren ordenagailuan izateko.
Jada baduzu ez duzu ezer egin behar. Bestela, ez baduzu eta powershell bidez instalatu nahi baduzu hurrengoa egin:
Instalatzeko:
```
winget install --id Microsoft.VisualStudioCode -e --source winget
```
Ondoren instalatuta dagoela begiratu:
```
code --version
```
Hurrengoa edo antzeko bat agertu behar da:
```
1.95.2
```
Ezabatu egin behar baduzu:
```
winget uninstall --id Microsoft.VisualStudioCode -e
```
 
#### 5. Git
Git gure GitHub-eko biltegiarekin lan egiteko balio du. Bere [webgunetik](https://git-scm.com/install/) instalatu daiteke, VSCode bezala;

Ahala ere, powersheletik egin nahi baduzu:
Instalatzeko:
```
winget install --id Git.Git -e --source winget
```
Egoki instalatu den jakiteko:
```
git --version
```
Hurrengoa edo antzekoa agertu behar da:
```
git version 2.47.0.windows.1
```
Desinstalatu behar baduzu:
```
winget uninstall --id Git.Git -e
```
#### 6. Docker
<p align="justify"> Odoo exekutatu ahal izateko, Docker beharrezkoa izango dugu. Docker Desktop instalatzea gomendatzen dugu. </p>

Instalatzeko (PowerShell administratzaile baimenekin):

```
winget install -e --id Docker.DockerDesktop
```

Instalazioa amaitzean, ordenagailua berrabiarazi beharko duzu. Ondoren, Docker ondo dagoen egiaztatzeko:

```
docker --version
```

Docker instalatu ondoren, proiektua abiarazi behar duzu. Normalean, proiektuak fitxategi bat izango du docker-compose.yml izenekoa.

Hau da egin beharko duzuna:
1. Kodea jaitsi: Git bidez proiektua ordenagailura jaitsi (git clone ...).

2. Karpetan sartu: Terminalean proiektuaren karpetara sartu.

3. Docker piztu: Hurrengo komandoa exekutatu:

```
docker-compose up -d
```

## Odooren erabilera

### 1. Justifikazioa

Gure "Pisukide" proiekturako ERP sistema bat integratzea ezinbestekotzat jo dugu arrazoi nagusi hauengatik:

* **Datuen Zentralizazioa:** Pisukideen, pisuetako informazioaren eta ordainketen datuak leku bakar eta seguru batean kudeatzeko. Frontend-a (Laravel/React) erabiltzailearen esperientziarako den bitartean, ERPa datuen "bihotza" da.
* **Prozesuen Kudeaketa:** Ez dugu hutsetik programatu behar, gure kasuan gastu modulua erabili dugu. ERPak modulu horiek garatuta dakartza, eta horrek garapen denbora asko aurrezten digu.
* **Segurtasuna eta Egonkortasuna:** Datu kritikoak kudeatzeko, frogatuta dagoen eta segurua den sistema bat erabiltzea hobe da guk hutsetik sortutako (eta agian akatsak dituen) backend bat baino.


### 2. Zergatik Odoo?

Merkatuan dauden beste aukeren aurrean, **Odoo** aukeratu dugu arrazoi hauengatik:

* **Kode Irekia (Open Source):** Odoo-k "Community" bertsioa du, guztiz doakoa eta kode irekikoa dena. Honek lizentzia garestiak ordaindu gabe proiektua garatzeko aukera ematen digu, ikasle proiektu baterako aproposa.
* **Modularitatea:** Ez dugu sistema osoa instalatu beharrik. Bakarrik behar ditugun "Apps" edo moduluak instalatu ditzakegu. Honek sistema arinagoa egiten du.
* **Integrazio Erraza (API):** Odoo-k kanpo konexioetarako API oso indartsua du. Honek ahalbidetzen digu gure Laravel eta React aplikazioa Odoorekin komunikatzea arazorik gabe (adibidez, erabiltzaile bat webgunetik sortzean, Odoon ere sortzea).
* **Teknologia Modernoa:** Python eta PostgreSQL erabiltzen ditu, gaur egun ikasteko errazak diren teknologiak dira.
* **Erabilerraztasuna:** Interfaze garbia eta intuitiboa dauka, beste ERP tradizional batzuk baino askoz modernoagoa.

### 3. SW eta HW Eskakizunak

Ordenagailuan garapena egiteko gutxienekoak:

#### Hardware Eskakizunak (HW)

Ordenagailuan garapena egiteko gutxienekoak:

* **Prozesadorea (CPU):** 64-bit-eko prozesadorea, birtualizazioa onartzen duena (Intel Core i5 edo baliokidea gomendatzen da).
* **RAM Memoria:**
    * *Gutxienez:* 8 GB (Odoo eta sistema eragilea justu ibiliko dira).
    * *Gomendatua:* 16 GB edo gehiago (Dockerrek memoria asko kontsumitzen du).
* **Disko Gogorra:** SSD diskoa gomendatzen da biziki, datu-basearen irakurketa azkarra izateko. Gutxienez 50 GB libre egon behar dira Docker irudietarako eta datuetarako.


#### Software Eskakizunak (SW)

Ingurunea prestatzeko beharrezkoa den softwarea:

* **Sistema Eragilea:**
    * Windows 10/11 (64-bit) -> *WSL 2 (Windows Subsystem for Linux) aktibatuta egotea beharrezkoa da.*
    * Linux (Ubuntu, Debian...)
    * macOS.
* **Birtualizazioa:** BIOS/UEFIan birtualizazioa aktibatuta (Virtualization Technology - VT-x edo AMD-V).
* **Docker Desktop:** Odoo edukiontziak kudeatzeko eta altxatzeko tresna.
* **Nabigatzailea:** Google Chrome, Firefox edo Edge (Odoo-ren interfazea ikusteko).
* **Datu-basea (Docker barruan):** PostgreSQL (normalean `docker-compose.yml` fitxategian definituta dator eta automatikoki instalatzen da edukiontzi bezala, ez da Windowsen bertan instalatu behar).

## Irisgarritasuna

Webgunearen garapenean irisgarritasuna funtsezko zutabe gisa tratatu da, pertsona guztiek, beren gaitasunak edozein direla ere, edukiera sartzeko aukero izan dezaten.
Helburu hori lortzeko, EqualWeb irisgarritasun-widget aurreratua inplementatu dugu.

Araudia eta Estandarrak gure proiektuak WCAG 2.1 jarraibideak betetzen du, zehazki AA mailakoa bilatuz. EqualWeb tresnaren bidez, webgunearen kodea eta interfazea automatikoki egokitzen dira nazioarteko estandar horietara.
Hartutako neurriak, EqualWeb auekra pertsonalizatuen menu bat eskaintzen dio erabiltzaileari, lau arlo nagusitan banatuta:

- **Ikusmen-egokitzapenak:**
  - **Konstrasteak:** "Kontraste iluna", "Kontraste argia" eta "Monokromo" moduak aktibatu daitezke, daltonismoa edo ikusmen-nekea dutenentzat.
  - **Testua eta Tamaina:** "Lupa" eta "Botoiak handitzeko" aukera, eta "Letra irakurgarria" (letra-formatua aldatu egiten da).
  - **Kolore pertsonalizatuak:** Erabiltzaileak webguneko fondoen eta testuen koloreak alda ditzake bere beharren arabera.
- **Nabigazioa eta Motrizitatea:**
  - **Teklatu bidezko nabigazioa:** Sagua erabili ezin dutenentzat, webgune osoa Tab eta Enter teklak erabiliz nabigatu daiteke ("Navegación con teclado").
  - **Teklatu birutala:** Gailu periferiko fisikoak erabiltzeko zailtasunak dituztenentzat.
- **Laguntza Kognitiboa:**
  - **Elementuak nabarmendu:** "Estekak nabarmendu" eta "Izenburuak nabarmendu" aukerek webgunearen egitura hobeto ulertzen laguntzen dute.
  - **Irakurketa modua:** Distrakzioak ezabatzen ditu ("Modo legible") eta arreta edukian zentratzen du.
  - **Orriaren laburpena:** Edukiaren ulermena errazteko.
- **Entzumena eta Edukia:**
  - **Testu irakurlea:** Pantailako edukia ahots bidez irakurtzen du itsuentzat.
  - **Irudien deskribapena:** Irudien testu alternatiboak kudeatzen ditu.
  
### WCAG 2.1-AA Mailaren justifikazioa

Gure aplikazioak WCAG 2.1 (Web Content Accessibility Guidelines) estandarraren AA maila betetzen duela bermatzeko, EqualWeb irtenbide teknologikoa integratu dugu.  
Tresna honek webgunearen kodea eta interfazea automatikoki moldatzen ditu, AA mailak eskatzen dituen arrakasta-irizpide (Success Criteria) kritikoenak betetzeko.

Honela justifikatzen dugu maila hori lortu izana:



### 1. **Hautemangarria**

Informazioa eta interfazea erabiltzaileek hautemateko moduan aurkeztu behar dira.

- **Kontrastea**
  - **Baldintza:** Testuaren eta atzealdearen arteko kontraste-erlazioa gutxienez 4.5:1 izatea.
  - **Gure aplikazioan:** Widgetak *Contraste oscuro*, *Contraste claro* eta *Monocromo* moduak eskaintzen ditu. Gainera, *Color personalizado* atalak aukera ematen dio erabiltzaileari koloreak eskuz doitzeko, irakurgarritasun maximoa bermatzeko.

- **Testuaren Tamaina Aldatzea**
  - **Baldintza:** Testua %200raino handitu ahal izatea edukia edo funtzionaltasuna galdu gabe.
  - **Gure aplikazioan:** *Lupa*, *Lupa de texto* eta *Botones para ampliar* funtzioek aukera ematen diote erabiltzaileari testua eskalatzeko, ikusmen baxua duten pertsonen beharretara egokituz.

- **Eduki ez-testuala**
  - **Baldintza:** Irudiek testu alternatiboa izatea.
  - **Gure aplikazioan:** *Descripción de imágenes* funtzioak eta *Lector de texto*-k bermatzen dute pantaila-irakurleek irudien informazioa transmititu ahal izatea.



### 2. **Erabilgarria**

Interfazearen osagaiak eta nabigazioa erabilgarriak izan behar dira.

- **Teklatu bidezko Sarbidea**
  - **Baldintza:** Funtzionalitate guztiak teklatuaren bidez erabili ahal izatea.
  - **Gure aplikazioan:** *Navegación con el teclado (Motorizada)* aukerak webgunearen egitura egokitzen du Tab eta Enter bidez nabigatzeko. Gainera, *Teclado virtual* eskaintzen du periferiko fisikoak erabili ezin dituztenentzat.

- **Fokuaren Ikusgarritasuna**
  - **Baldintza:** Teklatuaren fokuak (kurtsorea non dagoen) beti ikusgai egon behar du.
  - **Gure aplikazioan:** Teklatu bidezko nabigazioa aktibatzean, widgetak fokua nabarmentzen du, eta *Resaltar enlaces* eta *Resaltar títulos* aukerek orientazioa errazten dute.

- **Krisi eta Konbultsioak Prebenitzea**
  - **Gure aplikazioan:** *Silenciar medios* aukerak bideoak eta audioak gelditzeko aukera ematen du, distrakzioak edo estimulu arriskutsuak saihestuz.



### 3. **Ulergarria**

Informazioak eta erabilerak ulergarriak izan behar dute.

- **Irakurgarritasuna eta Egokitzapen Kognitiboa**
  - **Gure aplikazioan:** *Letra legible* (Letra irakurgarria) funtzioak webguneko tipografia aldatzen du dislexia duten pertsonentzat diseinatutako formatu batera.
  - *Resumen de página* aukerak edukiaren sintesia eskaintzen du, arreta edo ulermen arazoak dituzten erabiltzaileei laguntzeko.

## Erabilgarritasuna

Gure plataformaren erabilgarritasuna funtsezkoa da, pisu partekatu baten kudeaketa (gastuak eta zereginak) askotan gatazka iturri izan baitaiteke. Helburua marruskadurak ezabatzea da, interfaze intuitibo eta bizkor baten bidez. Jakob Nielsenen heuristikoetan eta UX printzipioetan oinarrituta, honako neurri hauek hartu ditugu:

- Nabigazio Argia eta Arbel Zentralizatua (Dashboard):
  - Hasierako pantailak laburpen bisual bat eskaintzen du: "Nire zereginak" (Egiteko edo egiten darenak). Honek nabigazioa murrizten du, informazio kritikoena klik bakarrera baitago.
  - Menu Sinplifikatua: Egitura logikoan banatu dugu aplikazioa: Gastuak, Zereginak, pisuak eta aukeratutako pisua. Ez dago azpi-menu konplexurik; funtzio nagusiak beti daude eskura, batez ere gailu mugikorretan erabiltzeko pentsatuta.
- Gastuen Kudeaketaren Argitasuna:
  - Kolore-kodeen erabilera: Interfazeak koloreak erabiltzen ditu informazioa azkar prozesatzeko (adibidez, berdea "jasotzeko duzun dirua" eta gorria "zor duzuna"). Honek karga kognitiboa murrizten du; erabiltzaileak ez du kalkulurik egin behar, begiratu batean ulertzen du egoera.
- Zereginen Kudeaketa eta Interakzioa:Etxeko lanen banaketa modu eraginkorrean kudeatzeko, zerrenda tradizionaletatik haratago joan gara, egutegi interaktibo bat inplementatuz.
  - Ikuspegi Tenporala: Egutegiak aukera ematen die pisukideei zereginak denboran banatuta ikusteko (adibidez: "Nork garbitu behar du ostiralean?"). Honek planifikazioa errazten du eta "ahazturak" saihesten ditu.
  - Egoeraren Feedback Bisuala (Kolore-kodeak): Zereginen egoera identifikatzeko ez da testurik irakurri behar; koloreen bidezko sistema semantikoa erabiltzen dugu:
    - Berdea: Zeregina osatuta dago.
    - Gorria: Zeregina egiteko dago.
    - Urdina: Zeregina egiten dago.
      
- Irisgarritasuna:
  - EqualWeb integrazioa: Kolore-kodeetan oinarritzen garenez, irisgarritasun-widgetak aukera ematen du kontrasteak doitzeko edo daltonismoa duten erabiltzaileentzat koloreak egokitzeko, infromazioa (zereginen egoera) inork gal ez dezan.

## Posizionamendua:

### 1. **SEO**
Bilatzaileetan modu organikoan agertzeko, gure aplikazioan <i>app.blade.php</i> fitxategian aldaketa haauek egin dira:
- Hitz Gakoak: Etiketa espezifikoak definitu ditugu gure nitxoarekin bat egiteko. Erabilitako hitzak hauek dira: "pisukide, pisua kudeatu, gastuak, garbiketa, ikasle pisuak, elkarbizitza". Honek Google-i adierazten dio webgunea kudeaketari eta elkarbizitzari buruzkoa dela.
- Meta-deskribapena: Deskribapen labur eta erakargarria txertatu dugu: "Zure pisua kudeatzeko aplikazioa. Antolatu gastuak, zereginak eta erosketak zure pisukideekin erraz.". Testu hau da bilaketa-emaitzetan agertzen dena, klik egitera bultzatuz.
- Izenburu Dinamikoak: <title inertia> etiketaren bidez, orri bakoitzak (Gastuak, Zereginak, etab.) bere izenburua izango du, indexazioa hobetuz.
- Errendimendu Optimizazioa: Googlek webgune azkarrak saritzen dituenez, letra-tipoak aurrez kargatzen dira.
- Hizkuntzaren definizioa: HTML etiketan hizkuntza dinamikoki definitzen da (lang="{{ ... app()->getLocale()) }}"), bilatzaileek edukia zein hizkuntzatan dagoen ondo uler dezaten.
  
### 2. **SMO**
Sare sozialetan estekak partekatzean itxura profesionala izatea ezinbestekoa da konfiantza sortzeko. Horretarako, Open Graph eta Twitter Cards protokoloak inplementatu ditugu app.blade.php fitxategian:
- Txartel Sozialak: Esteka bat partekatzean, ez da URL soila agertzen. Izenburua ("Zure pisua kudeatzeko aplikazioa...") eta deskribapena automatikoki kargatzen dira.
- Irudi Pertsonalizatua: Irudi erakargarri bat diseinatu dugu (SMOPhoto.png) espresuki sare sozialetarako. Irudi honek giro atsegina erakusten du, pisukide talde bat afaltzen eta ondo pasatzen, "Find your ideal flatmate" leloarekin, aplikazioaren helburu soziala indartzeko.
- Plataforma anitzeko bateragarritasuna: og:type, og:title eta twitter:card etiketek bermatzen dute aurrebista ondo ikusiko dela bai iPhonen (iMessage), bai Androiden eta baita ordenagailuko nabigatzaileetan ere.
